🔒对称加密

Symmetric Encryption

对称加密是最古老、最直观的加密方式。加密和解密使用同一个密钥，就像一把钥匙既能锁门又能开门。发送方用密钥将明文加密为密文，接收方用同一把密钥将密文解密还原为明文。

工作原理

发送方和接收方必须事先通过安全渠道共享密钥。加密算法接受明文和密钥作为输入，输出密文；解密算法接受密文和同一密钥，输出明文。安全性完全依赖于密钥的保密——如果密钥泄露，所有通信内容都会被破解。

常见算法

算法密钥长度分组大小特点
DES56 位64 位1977 年标准，现已不安全（密钥太短，可暴力破解）
3DES112/168 位64 位DES 的增强版，三次加密，但速度慢
AES128/192/256 位128 位当前最广泛使用的对称加密标准，安全高效
SM4128 位128 位中国国家密码标准，用于无线局域网等领域
ChaCha20256 位流密码现代流密码，速度快，用于 TLS 1.3 等协议

工作模式

分组密码本身只能加密固定大小的数据块。为了加密任意长度的消息，需要结合工作模式：

  • ECB（电子密码本）：每个块独立加密，相同明文块产生相同密文块，不安全
  • CBC（密文分组链接）：每个明文块先与前一个密文块异或，再加密，需要初始向量 IV
  • CTR（计数器模式）：将分组密码转为流密码，支持并行处理
  • GCM（Galois/计数器模式）：在 CTR 基础上增加认证功能，同时保证机密性和完整性
优点：加解密速度快（比非对称加密快 100-1000 倍），适合大量数据加密。
缺点：密钥分发困难——双方必须事先安全共享密钥。N 个人通信需要 N(N-1)/2 个密钥。

数学表示

E(K, M) = C   |   D(K, C) = M

其中 K 为密钥，M 为明文，C 为密文，加密函数 E 和解密函数 D 满足 D(K, E(K, M)) = M。

🔓非对称加密

Asymmetric Encryption / Public-Key Cryptography

非对称加密使用一对数学上关联但不相同的密钥：公钥（public key）和私钥（private key）。公钥可以公开分发，私钥必须保密。公钥加密的内容只有对应私钥能解密，反之亦然。

核心思想

类比：一个带锁的信箱。任何人都可以通过投信口（公钥）把信放进信箱（加密），但只有持有钥匙的人（私钥）能打开信箱取出信件（解密）。这完美解决了对称加密中"密钥分发"的痛点。

主要算法

算法数学基础典型密钥长度特点
RSA大整数分解2048-4096 位历史最悠久，应用最广，加解密较慢
ECC / ECDSA椭圆曲线离散对数256-384 位密钥更短、速度更快，移动端首选
SM2椭圆曲线（自定义曲线）256 位中国国家密码标准，用于金融等领域
ElGamal离散对数2048+ 位基于 Diffie-Hellman 扩展，支持同态性质
Lattice-based格问题变化后量子密码候选，抗量子计算攻击

RSA 算法流程

  1. 随机选择两个大质数 p 和 q
  2. 计算 n = p × q，计算欧拉函数 φ(n) = (p-1)(q-1)
  3. 选择 e 使得 1 < e < φ(n) 且 gcd(e, φ(n)) = 1
  4. 计算 d = e⁻¹ mod φ(n)（模逆元）
  5. 公钥 = (n, e)，私钥 = (n, d)
  6. 加密：c = mᵉ mod n  |  解密：m = cᵈ mod n
应用场景：HTTPS/TLS 握手交换对称密钥、电子邮件加密（PGP）、数字货币钱包、SSH 远程登录认证。

🧩哈希函数

Hash Function

哈希函数将任意长度的输入映射为固定长度的输出（哈希值/摘要）。这个输出就像数据的"指纹"——即使输入只改变一个比特，输出也会完全不同（雪崩效应）。

密码学哈希的三个核心特性

  • 抗原像性（单向性）：给定哈希值 h，在计算上不可能找到 x 使得 hash(x) = h。即"不可逆推"。
  • 抗第二原像性：给定 x₁，在计算上不可能找到不同的 x₂ 使得 hash(x₁) = hash(x₂)。
  • 抗碰撞性：在计算上不可能找到任意两个不同的输入 x₁ ≠ x₂ 使得 hash(x₁) = hash(x₂)。

常见算法

算法输出长度状态
MD5128 位已破解，存在碰撞攻击，不应再使用
SHA-1160 位已破解，Google 在 2017 年实现了首次碰撞攻击
SHA-256256 位当前最广泛使用的安全哈希，比特币核心算法
SHA-3 (Keccak)可变新一代标准，与 SHA-2 结构完全不同
SM3256 位中国国家密码标准，用于数字签名和验证
BLAKE3256 位现代高性能哈希，比 SHA-256 快数倍

实际应用

  • 密码存储：网站不存储明文密码，而是存储 hash(password + salt)。登录时比对哈希值，即使数据库泄露也不会暴露原始密码。
  • 文件完整性校验：下载文件时提供 SHA-256 校验和，确保文件在传输过程中未被篡改。
  • 区块链：每个区块包含前一个区块的哈希，形成不可篡改的链。比特币挖矿本质上是在寻找满足特定条件的哈希值（工作量证明）。
  • 数字签名：先对消息取哈希，再对哈希值签名（而非对整个消息签名），大幅提高效率。
加盐（Salt）：为每个用户的密码附加一个随机值再进行哈希。即使两个用户密码相同，加入不同盐后哈希值也完全不同，有效防御彩虹表攻击。

🎭零知识证明

Zero-Knowledge Proof (ZKP)

零知识证明是一种协议，允许一方（证明者 Prover）向另一方（验证者 Verifier）证明自己知道某个秘密，而不透露秘密本身的一丁点信息。验证者最终只确信"证明者确实知道这个秘密"，而对秘密内容一无所知。

三个基本性质

  • 完整性（Completeness）：如果证明者真的知道秘密，验证者一定会被说服。
  • 可靠性（Soundness）：如果证明者不知道秘密，他几乎不可能欺骗验证者（欺骗概率可随重复次数指数级下降）。
  • 零知识性（Zero-Knowledge）：验证者除了"证明者知道秘密"这一事实外，学不到任何关于秘密本身的信息。

经典例子：阿里巴巴的山洞

阿里巴巴知道山洞暗门的咒语。他想向 Victor 证明自己知道咒语，但不透露咒语内容。山洞有出口 A 和 B，中间暗门需要咒语打开。

流程：Victor 在洞外等待 → 阿里巴巴从 A 或 B 任意进入 → Victor 随机要求他从 A 或 B 出来 → 如果阿里巴巴知道咒语，总能在必要时穿过暗门从正确出口出来 → 如果不知道，只有 50% 蒙对概率。重复 20 轮后骗子成功率低于百万分之一。

主要类型

类型特点代表系统
交互式 ZKP证明者和验证者需要多轮交互经典图同构证明
非交互式 ZKP (NIZK)证明者只生成一个证明，验证者可独立验证zk-SNARKs
zk-SNARKs证明短小、验证快、需可信设置Zcash、zkSync
zk-STARKs无需可信设置、抗量子、证明较大StarkNet、StarkEx

应用前景：区块链隐私交易（隐藏金额和地址）、身份验证（证明成年而不透露生日）、合规审计（证明资产充足而不公开账户）、可验证计算（外包计算并验证结果正确性）。

✍️数字签名

Digital Signature

数字签名是非对称加密的"逆向"应用——用私钥签名，用公钥验证。它提供三个核心保证：真实性（来源可信）、完整性（内容未被篡改）、不可否认性（签名者无法抵赖）。

签名与验证流程

签名：发送方对消息计算哈希 → 用私钥对哈希值加密 → 生成签名 → 将消息和签名一起发送。
验证：接收方对消息重新计算哈希 → 用发送方公钥解密签名 → 比对两个哈希值。一致则验证通过。

Sign(M, sk) = σ   |   Verify(M, σ, pk) = True/False

常见算法

算法数学基础应用
RSA-PSS大整数分解TLS 证书、旧版 DNSSEC
ECDSA椭圆曲线比特币、以太坊交易签名
Ed25519 (EdDSA)扭曲爱德华曲线SSH 密钥、现代 TLS、Solana
SM2-Sign椭圆曲线（自定义曲线）中国金融 IC 卡、电子认证
Schnorr离散对数比特币 Taproot 升级
签名 ≠ 加密：虽然都使用公私钥对，但方向相反。加密是用公钥加密、私钥解密；签名是用私钥签名、公钥验证。两者可以结合使用：先签名再加密，同时保证机密性和认证性。

🔗区块链中的密码学

Cryptography in Blockchain

区块链并不是"一种"密码学技术，而是多种密码学工具的系统性组合。理解区块链，就是理解这些密码学组件如何协同工作。

密码学组件在区块链中的角色

组件在区块链中的作用
哈希函数 (SHA-256)区块链接（每个区块含前一区块哈希）；工作量证明挖矿；默克尔树构建交易摘要；地址生成
数字签名 (ECDSA)交易签名与验证——只有私钥持有者才能花费对应地址的资产
公钥密码学地址派生：公钥 → 哈希 → 编码 → 地址。拥有私钥即拥有资产控制权
默克尔树 (Merkle Tree)高效验证某笔交易是否包含在区块中（SPV 轻节点验证），无需下载完整区块
零知识证明隐私链（Zcash）隐藏交易金额和地址；Layer2 扩容（zk-Rollup）压缩验证数据

比特币交易验证流程

  1. Alice 创建交易：指定输入（之前收到的 UTXO）、输出（Bob 地址 + 金额）、找零
  2. Alice 用她的私钥对交易进行数字签名，证明她有权花费这些 UTXO
  3. 交易广播到网络，节点验证签名和输入有效性
  4. 矿工将有效交易打包进候选区块，开始工作量证明（不断尝试 nonce 使区块哈希满足难度目标）
  5. 矿工找到有效 nonce → 广播区块 → 其他节点验证 → 加入区块链
  6. 后续区块在该区块之上构建，提供确认数保障不可逆性

默克尔树原理

区块中所有交易的哈希两两配对，逐层向上哈希，最终形成一个唯一的默克尔根（Merkle Root）存入区块头。要验证某笔交易是否在区块中，只需提供该交易到根的路径（约 log₂N 个哈希），无需下载全部交易。这让手机钱包等"轻节点"成为可能。

为什么区块链"不可篡改"？每个区块的哈希依赖于其包含的交易数据和前一个区块的哈希。修改任何一个区块的任何数据都会改变其哈希，导致所有后续区块的哈希失效。攻击者必须重新完成从被篡改区块开始的所有工作量证明，这在算力上几乎不可能。
量子计算威胁：大规模量子计算机若能运行 Shor 算法，将能高效分解大整数和求解离散对数，从而破解 RSA 和 ECDSA。这促使密码学界加速推进后量子密码学（PQC）标准化。区块链社区也在探索抗量子签名方案。
